======================================================================
バージョン番号ルール↓
※ なるべく一般的なルールに合わせていますが、これが世界共通で唯一というわけではありません。

{メジャー番号}.{マイナー番号}.{ビルド番号}

  -> メジャー番号: マイナー番号が 9 の次、もしくは大幅アップデート（≒作り直し）
  -> マイナー番号: 機能追加系、普段のアップデートはこれを +1 する
  -> ビルド番号: いちいちカウントするのがダルいので、リリースするときは基本的にゼロにする
    （マイナー番号を +1 してビルド番号はリセット、という考え方です）

======================================================================
バージョンアップ手順↓

1. ソースコードやシーン、アセットの修正を完了させる

  -> タイトル画面のバージョン番号を更新して下さい。
  -> 基本的にリリースした後は次のリリースが決まるまで編集しないで下さい。


2. ビルドする

  -> Windows [x86_64]

    基準フォルダー（どこでもいい）
      /Windows
        /x64
          / 以下に WoS.exe をビルド

  -> Mac [Universal]

    基準フォルダー（Windowsと同じところ）
      /Mac
        / 以下に WoS.app をビルド

  -> ビルドした後、最終コミット＆プッシュを行い、その最終コミットにタグを [1st リリース] のようにつけてプッシュして下さい。
  -> その後、GitBucketのReleasesで他と同様にタグ（Release）に対してバージョン番号と更新内容を記入して下さい。


3. ビルドしたフォルダーをzipで圧縮

  -> WindowsはWindowsフォルダー、MacはWoS.appをそれぞれzip形式で圧縮して下さい。


3. Readme-x-x.0.txt をコピーして編集

  -> タイトルのバージョンを更新して下さい。
  -> 追加した内容を最後の更新履歴のところに追記して下さい。


4. ReadmeをビルドフォルダーのWindowsとMac以下にそれぞれコピーする

5. 創作活動部のGoogleドライブにログイン（いつもの）してアップロードする

  /創作活動部/ホームページ公開用/ソフトウェア/World of Snowplows/x.x.0 [ほにゃららリリース]/ 以下
  にビルドしたフォルダーをコピー
  
  -> アップロード完了後、WindowsフォルダーとMacフォルダーの共有リンクをそれぞれ取得しておきます。


6. サイトを更新する

  -> WordPressにログイン（いつもの）して、作品->「World of Snowplows」の編集画面に入ります。
  -> 記事本文の先頭に更新した旨を追記して下さい。
  -> 記事本文の末尾にあるダウンロードリンクURL（Windows/Mac）を更新して下さい。
  -> 記事本文の末尾にあるバージョン番号を更新して下さい。
